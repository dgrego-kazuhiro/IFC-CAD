# 日本住宅向け CAD 仕様（RoomGrid CAD）

## 結論

「部屋ベース CAD」ではなく、**表向きはグリッド・区画ベース CAD、内部的には部屋ベース管理 CAD** にする。

特に日本の一般住宅向けでは、最初から完全自由な内法線入力にするより、**910mm / 455mm グリッドにスナップして部屋区画を描く**仕様が自然である。

### 推奨コンセプト

| 観点 | 内容 |
|------|------|
| 入力 | 910mm / 455mm グリッド上で部屋区画を描く |
| 管理 | Room / Boundary / Wall / WallFace として部屋中心に管理 |
| 生成 | 境界から壁・柱・開口・内法面積を自動生成 |
| 調整 | 必要に応じて自由寸法やスナップ解除も可能 |

> 一言で言うと、**芯・グリッドで描けるが、内部では部屋・壁・面積が自動管理される CAD**。

---

## 1. 初期モード

日本住宅向けの初期モードは以下とする。

- **モード名**：日本住宅モード / 尺モジュールモード
- **主グリッド**：910mm
- **補助グリッド**：455mm
- **スナップ**：ON
- **入力基準**：壁芯 / 区画芯
- **壁厚初期値**：105mm
- **面積表示**：畳数 + ㎡ + 推定内法面積

### 表示

- 910mm グリッド：濃い線
- 455mm グリッド：薄い線
- 柱候補：グリッド交点に小さく表示
- 仮壁：薄いグレー
- 部屋：淡い色で塗り分け

---

## 2. 最初に描くもの

ユーザーが最初に描くのは、**厳密な壁ではなく、部屋区画**である。

- 矩形部屋
- 閉じたポリライン部屋
- グリッド範囲選択

ただし、内部的には以下のように分ける。

| 種別 | 用途 |
|------|------|
| 入力線 | 区画線 / 壁芯候補 |
| 面積計算線 | 生成された内法線 |
| 壁中心線 | 区画線から生成 |
| 壁実体 | 壁中心線 + 壁厚から生成 |

**最初に描いた線をいきなり「内法線」としない**。日本住宅向けでは、入力線 ≒ 壁芯・柱芯に乗る区画線、として扱うのが自然。

---

## 3. 部屋作成時の表示

部屋を描いている最中に、以下をリアルタイム表示する。

- 幅：3,640mm（4P）
- 奥行：2,730mm（3P）
- 芯々面積：9.94㎡
- 推定内法面積：9.28㎡
- 畳数：約6.0畳

ここで、

- `1P = 910mm`
- `0.5P = 455mm`

として表示する。設計者には、mm だけでなく **P数・畳数** を出すと分かりやすい。

---

## 4. 壁の扱い

壁は、部屋作成時点では**確定壁ではなく、仮壁**として表示する。

- 部屋区画線：濃い線
- 仮壁：薄いグレーの帯
- 内法線：必要時だけ表示
- 壁中心線：詳細表示で表示
- 外法線：外壁確認時に表示

壁は内部的には、部屋境界から生成する。

```
RoomBoundary + WallType + Thickness + Level → Wall
```

壁の種類は境界ごとに持たせる。

- 外壁
- 間仕切壁
- 耐力壁
- 防火壁
- ガラス間仕切
- 壁なし
- 仮想境界

---

## 5. 境界データの基本構造

内部データは、少なくとも次のように分ける。

```ts
type Room = {
  id: string;
  name: string;
  levelId: string;
  boundaryLoopId: string;
  targetArea?: number;
  actualInnerArea: number;
};

type BoundarySegment = {
  id: string;
  start: Vec2;
  end: Vec2;
  leftRoomId?: string;
  rightRoomId?: string;
  boundaryType: "external" | "shared" | "virtual" | "open";
  wallTypeId?: string;
  fixRule: "keepRoomArea" | "keepGrid" | "keepExterior" | "keepWallCenter";
};

type Wall = {
  id: string;
  sourceBoundaryId: string;
  wallTypeId: string;
  axis: Curve2D;
  thickness: number;
  baseLevelId: string;
  height: number;
  isGenerated: boolean;
};

type WallFace = {
  id: string;
  wallId: string;
  roomId?: string;
  side: "left" | "right" | "exterior";
  finishTypeId?: string;
};
```

特に重要なのは、**Room / BoundarySegment / Wall / WallFace を分けること**。

## 6. エッジ長が違う部屋の接続

長さの違う辺どうしが一部だけ接続する場合は、**エッジ全体を共有にしない**。

- 重なる区間だけ共有境界
- 残りエッジ1は外部境界
- 残りエッジ2も外部境界

つまり、必ず境界を分割する。

```
元エッジ
  ↓
残りエッジ1 + 共有エッジ + 残りエッジ2
```

共有エッジだけに間仕切壁を割り当て、残りは外壁または未接続境界にする。

---

## 7. 壁作成時に「何を守るか」を選べるようにする

壁を生成するときに一番重要なのは、**何を固定するか**。

UI では、境界や壁を選択したときに以下を選べるようにする。

> **この壁の変更時に守るもの：**
> - 部屋面積を守る
> - グリッド / 壁芯を守る
> - 建物外形を守る
> - 構造芯を守る

| モード | 初期値 |
|--------|--------|
| 日本住宅モード | グリッド / 壁芯を守る |
| 自由ゾーニングモード | 部屋面積を守る |

---

## 8. モードを分ける

1つの操作体系だけで全部やろうとすると破綻する。以下のモードを用意する。

### A. 自由ゾーニングモード

- **目的**：初期検討・施主打合せ・AI プラン生成
- **入力**：自由な部屋ブロック
- **基準**：内法または目標面積
- **表示**：面積・畳数・隣接関係
- **壁**：仮壁

### B. 日本住宅グリッドモード

- **目的**：一般住宅の間取り作成
- **入力**：910 / 455mm グリッドスナップ
- **基準**：壁芯 / 区画芯
- **表示**：P数・畳数・推定内法面積
- **壁**：仮壁から確定壁へ

### C. 実施設計モード

- **目的**：壁・柱・建具・納まりの調整
- **入力**：壁芯・柱芯・寸法指定
- **基準**：グリッド / 構造芯 / 外形
- **表示**：壁中心線・内法線・外法線・柱芯
- **壁**：確定壁

### D. ビル / 商業建築モード

- **目的**：面積計画・区画計画・テナント計画
- **入力**：部屋面積 + 構造グリッド
- **基準**：面積・コア・柱スパン
- **表示**：内法面積・貸室面積・共用部面積
- **壁**：区画壁・耐火壁・ガラス間仕切

---

## 9. 日本住宅向けの標準フロー

最も自然な流れ。

1. 尺モジュールモードを選択
2. 910mm / 455mm グリッドを表示
3. 部屋リストを作成
4. グリッド上に部屋区画を描く
5. P数・畳数・推定内法面積を確認
6. 部屋同士の接続を自動判定
7. 境界を分割・共有化
8. 仮壁を表示
9. 壁タイプを割り当てる
10. 柱候補・耐力壁候補を表示
11. 必要に応じてグリッド整合・寸法調整
12. 壁・柱・建具を確定
13. 3D モデル / IFC / 面積表を生成

## 11. スナップ仕様

日本住宅モードでは、スナップは強めにする。

| 操作 | 動作 |
|------|------|
| 通常 | 910mm / 455mm にスナップ |
| Shift | 水平・垂直固定 |
| Alt | 一時的にスナップ解除 |
| Ctrl | 寸法を数値入力 |
| Tab | 候補スナップ切替 |

### 対応スナップ

- グリッド点
- グリッド線
- 柱芯候補
- 壁芯候補
- 既存部屋境界
- 既存壁中心線
- 内法線
- 外法線
- 開口中心

---

## 12. 面積の扱い

面積は1種類だけにしない。最低限、以下を分ける。

- 芯々面積
- 推定内法面積
- 実内法面積
- 壁芯面積
- 法規面積
- 畳数換算

日本住宅では、部屋プロパティにこう出す。

```
寝室
  芯々寸法：3,640 × 2,730
  P数：4P × 3P
  畳数：約6.0畳
  推定内法：3,535 × 2,625
  推定内法面積：9.28㎡
```

---

## 13. 最終的な推奨仕様

### 日本住宅向け

- 910mm / 455mm グリッドで部屋区画を描く
- 入力線は壁芯・区画芯として扱う
- 部屋は Room として内部管理する
- 壁は RoomBoundary から仮生成する
- 面積は芯々・内法・畳数を分けて表示する
- エッジ接続時は重なり区間で境界分割する
- 壁作成時は「何を守るか」を選択可能にする
- 自由ゾーニングモードも別途用意する

### 一言でまとめると

> 日本向けの新しい建築 CAD は、最初から完全自由な内法線部屋 CAD にするより、**910mm / 455mm グリッドで部屋区画を描き、内部では部屋・境界・壁・面積を自動管理する仕様**がよい。
>
> つまり、従来の設計者には「**芯・グリッドで描ける CAD**」として見せ、内部では「**部屋ベース BIM**」として管理する。
